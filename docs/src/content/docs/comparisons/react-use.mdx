# vs react-use

`react-use` is a broad utility collection. Its responsive hooks are convenient, but reflow is purpose-built for responsive design and SSR.

## Why `useSyncExternalStore` matters

React introduced `useSyncExternalStore` so external subscriptions can behave correctly in concurrent rendering, Strict Mode, and SSR. reflow builds its responsive React adapter around that API.

That matters because media query and viewport hooks often fail in one of two ways:

- they read browser state too early during SSR, or
- they only repair the value in an effect after the first paint.

reflow avoids both by pairing a live store subscription with a stable `getServerSnapshot()`.

## Side-by-side

```tsx
// react-use style
import { useMedia } from "react-use";

const prefersDark = useMedia("(prefers-color-scheme: dark)", false);
```

```tsx
// reflow
import { useMediaQuery } from "reflow/react";

const prefersDark = useMediaQuery("(prefers-color-scheme: dark)", false);
```

The second example looks similar, but it is implemented on top of React's SSR-safe external store API.

## Migration steps

1. Replace `useMedia(query, initialState)` with `useMediaQuery(query, initialState)`.
2. Replace viewport hooks with `useViewport()`.
3. Replace repeated width checks with `createBreakpoints()` + `useBreakpoint()`.
4. Move color-scheme logic to `useColorScheme()` if you need persistence.
5. Add `ResponsiveProvider` when rendering on the server.
