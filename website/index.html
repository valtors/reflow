The problem

5 Packages. 22 KB. Zero Type Safety.

Most responsive stacks are still stitched together from unrelated hooks, brittle viewport listeners, and hand-rolled CSS helpers. You pay for the same signal multiple times, fight hydration bugs on first paint, and end up with responsive logic scattered across components, styles, and runtime guards.

One package handles media queries. Another does viewport width. A third pretends to do fluid typography. None agree on SSR.
You lose literal breakpoint inference, ship overlapping logic, and still end up guarding everything with typeof window.
fluidity-ts unifies breakpoints, clamp generation, container queries, prefers-* APIs, and SSR-first rendering into one coherent toolkit.

Use cases

Built for the places responsive code gets hardest.

SSR Apps

Read viewport hints during server rendering, choose the right breakpoint before the first paint, and hydrate into the same layout without flicker.

import { headers } from 'next/headers'
    import { resolveBreakpointFromHints } from 'fluidity-ts/server'

    const requestHeaders = await headers()
    const { width, breakpoint } = resolveBreakpointFromHints(requestHeaders)

    return <ResponsiveProvider serverWidth={width} initialBreakpoint={breakpoint}>...</ResponsiveProvider>

Design Systems

Keep tokens, spacing, and type scales in sync with one responsive source of truth instead of hand-tuning clamp values across every component.

import { fluidScale, responsiveStyle } from 'fluidity-ts/styles'

    export const typeScale = fluidScale([
      'body', 'title', 'display'
    ], { minPx: 16, maxPx: 72 })

    export const stack = responsiveStyle(system, 'gap', {
      xs: '1rem',
      lg: '2rem'
    })

Component Libraries

Build container-aware primitives that adapt to their own footprint, so cards, navs, and shells stay resilient wherever product teams compose them.

import { containerQuery, defineContainer } from 'fluidity-ts/styles'

    export const cardRoot = {
      ...defineContainer('card'),
      ...containerQuery('card', { minWidth: 420 }, {
        gridTemplateColumns: '1.4fr 1fr'
      })
    }

Dashboards

Tune dense, data-heavy interfaces by breakpoint so tables, filters, and side panels simplify automatically without another layer of resize plumbing.

import { useResponsiveValue } from 'fluidity-ts/react'

    const columns = useResponsiveValue({ xs: 1, md: 2, xl: 4 })

    <DataGrid
      columns={columns}
      density={columns === 1 ? 'compact' : 'comfortable'}
      showSidebar={columns >= 4}
    />
